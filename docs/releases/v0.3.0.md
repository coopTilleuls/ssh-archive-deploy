# v0.3.0

## Breaking configuration change

- Introduce configuration version 2 and reject version 1.
- Require consumers upgrading from `v0.2.5` or earlier to migrate their
  `deploy.yml` before changing the Action version.

## Remote capability diagnostics

- Add the read-only `doctor` command and Action mode to inventory required
  remote commands, GNU tar capabilities, and observable permissions.
- Return an `undetermined` verdict for tool versions that have not been
  qualified by the end-to-end compatibility matrix.
- Produce a versioned JSON result and a non-secret GitHub job summary without
  building a deployment archive.

## Packaging

- Allow ignored or untracked build products only through explicit `generated`
  declarations confined to a scope.
- Record generated inputs in the deployment manifest and support optional
  required-path checks.
- Reject unresolved Git LFS pointer content during both archive construction
  and validation.

## Validation coverage

- Add an integrated two-scope fixture with a generated Composer product and
  resolved Git LFS content.
- Verify complete manifest paths, sizes, and SHA-256 digests for the packaged
  artifact.

## Compatibility

- Keep the existing report, apply, and rollback command surface while changing
  the accepted configuration schema to version 2.
- Continue initial release support for Linux x86_64.
