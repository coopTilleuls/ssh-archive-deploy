## Security

- Pin every external GitHub Action used by the composite action and repository
  workflows to an immutable commit while retaining version comments for
  maintainable updates.
- Enforce full commit pinning for future workflow and composite-action changes
  with `zizmor` in the standard project checks.

## Release process

- Require substantive, versioned release notes to be committed and validated
  before a GitHub Release is created.

## Compatibility

- This patch does not change the CLI, configuration version, archive format, or
  supported Linux x86_64 runtime.
