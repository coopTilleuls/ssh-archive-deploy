# v0.3.2

## Compatibility

- Qualify GNU tar 1.35 for the remote capability contract used by `doctor`,
  alongside the existing GNU tar 1.34 qualification.
- Keep the compatibility decision based on an exact tested-version allowlist:
  later GNU tar versions remain `undetermined` until they are explicitly
  qualified.

## Validation coverage

- Run the complete SSH end-to-end suite against Debian Bookworm with GNU tar
  1.34 and Debian Trixie with GNU tar 1.35.
- Run the packaged PEX end-to-end suite and the local GitHub Action smoke test
  against GNU tar 1.35.
- Keep dedicated `mise` tasks for each qualified environment and retain the
  aggregate `test:e2e` task as the entry point for both suites.

## Operational impact

- Managed servers reporting GNU tar 1.35 can now receive a `compatible`
  verdict when all required commands and tar options are available.
- The CLI, configuration schema, archive format, and GNU tar 1.34 support are
  unchanged.
